# 🎯 나만의 스터디 원칙 & 지킬 일 (My Rules)

이 문서는 AI 에이전트 및 업무 자동화 스터디를 진행하며, 중도 포기하지 않고 실속 있는 성장을 이루기 위해 나 스스로와 타협하지 않을 개인 원칙을 기록하는 공간이다.

## 🏃‍♂️ 1. 스터디 참여 및 스프린트 원칙
- **유연하되 멈추지 않기:** 아무리 바쁜 주간이라도 아예 건너뛰지 않는다. 코드를 못 짜면 공식 문서 리딩이나 AI 툴 사용 후기(Docs) 한 줄이라도 남긴다.
- **철저한 기록 누적:** "머리로 이해한 것"은 공부가 아니다. 무조건 `docs/`나 `works/` 폴더에 흔적을 남겨 나만의 'AI 쿡북' 자산으로 만든다.
- **피드백에 열려있기:** 동료들이 주는 의견이나 피드백은 수용하고, 내 코드와 문서에 반영하려는 노력을 한다.

## 🛠️ 2. 개발 및 실습 원칙 (Works)
- **카피앤페이스트 금지:** AI가 짜준 코드를 생각 없이 복사 붙여넣기 하지 않는다. 한 줄 한 줄 왜 이렇게 작동하는지 이해하고, 주석을 달거나 내 스타일로 리팩토링한다.
- **실무/일상 적용 고민:** 단순히 튜토리얼 코드를 따라 하는 데 그치지 않고, "이걸 내 일상이나 프로젝트(예: Notion 연동, 반복 업무 등)에 어떻게 써먹을지" 최소 한 가지씩 확장 적용해 본다.
- **보안 철저:** API Key 노출은 절대 금지. 커밋 전 `.env`와 `.gitignore` 작동 여부를 항상 더블 체크한다.

## 🤖 3. AI 에이전트 스터디 가이드 규칙 (AI Agent Instructions)
이 저장소와 협업하는 모든 AI 에이전트(Antigravity 등)는 다음 지침을 반드시 준수해야 합니다.

- **대상 사이트 고정**: 사용자에게 기사나 아티클을 추천할 때는 반드시 **요즘IT (yozm.wishket.com)**, **디스콰이엇 (disquiet.io)** 이 두 가지 사이트에서 수집한 지식만 활용하여 추천합니다.
- **최신 4개월 이내 지식 추천**: 사용자에게 지식을 제안할 때는 **반드시 요청일 기준 최근 4개월 이내**에 발행/업데이트된 기사 또는 오픈소스만 추천합니다.
- **출처 링크 의무화**: 기사나 아티클을 인용/추천할 때는 사용자가 바로 접속하여 읽을 수 있도록 **반드시 해당 기사의 실제 원본 웹 링크(URL)** 또는 **GitHub 저장소 링크**를 함께 제공해야 합니다.
- **상세 설명 의무화**: 추천 후보군을 제안할 때는 단순히 제목과 링크만 나열하지 말고, **[1) 해당 주제가 다루는 핵심 기술/배경, 2) 스터디를 통해 배우는 엔지니어링 지식, 3) 사용자가 구현해야 할 실습 과제 시나리오]**를 각각 명확하게 설명해야 합니다.
- **기사 및 오픈소스 추천 후보의 독립적 분리**: 기사와 오픈소스를 억지로 엮어 매칭하지 않습니다. 추천을 요청받았을 때는 **[A. 최신 IT 기사 후보군]**과 **[B. 깃허브 트렌딩 오픈소스 후보군]**을 서로 완전히 무관한 독립된 주제들로 분리하여 제안합니다. 사용자가 기사를 고르면 해당 기사 학습 중심의 스터디를 셋업하고, 오픈소스를 고르면 해당 오픈소스 분석 및 활용 중심의 스터디를 셋업합니다.
- **중복 추천 원천 차단**: AI는 추천하기 전 반드시 `docs/` 및 `works/` 하위의 폴더명(예: `topic-[주제명]`)과 `study-plan.md` 파일에 기록된 이력 데이터를 전수 스캔해야 하며, 이미 학습한 기사의 URL/오픈소스 저장소 주소 혹은 이와 본질적인 주제가 겹치는 콘텐츠는 추천 후보 목록에서 영구 배제해야 합니다.
- **코드 작성 권한 차단**: 실습 세팅 시 뼈대 코드를 직접 제공하지 않습니다. AI는 이론 정리(`docs/`)와 실습 요구사항 명세서(`works/`)만 제공하며, 소스코드의 모든 작성과 설계는 사용자가 100% 직접 할 수 있도록 명세서만 배달합니다.
