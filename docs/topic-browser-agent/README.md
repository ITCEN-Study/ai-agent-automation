# 📅 [1주차 Docs] AI 자율 웹 브라우징 에이전트와 `browser-use`

이 문서에서는 AI가 직접 웹 브라우저를 띄워 사람처럼 요소를 클릭하고 검색하며 목표를 완수하는 **자율 브라우징 에이전트(Autonomous Web Agent)**의 구조와 작동 원리를 학습합니다.

* **연계 GitHub 오픈소스**: 🔗 [browser-use/browser-use GitHub Repository](https://github.com/browser-use/browser-use)

---

## 1. 자율 웹 브라우징 에이전트의 핵심 배경

기존의 단순 웹 스크래핑(Axios/Cheerio)은 정적 데이터 수집에 유리하지만, 다음과 같은 복잡한 비즈니스 자동화에는 대처할 수 없습니다:
1. 로그인 및 캡차(CAPTCHA) 통과
2. 동적 스크롤(Infinite Scroll) 및 모달 팝업 제어
3. 검색어 입력 및 페이지네이션 클릭 탐색

**자율 웹 에이전트**는 이러한 복잡한 다단계 브라우징 행위를 사람이 아닌 AI가 스스로 상태를 판단하고 브라우저 제어 라이브러리를 움직여 해결하도록 설계된 에이전트입니다.

---

## 2. browser-use의 아키텍처 및 구동 원리

`browser-use`는 AI 모델이 브라우저(Playwright/Puppeteer)를 효율적으로 다룰 수 있도록 돕는 대표적인 오픈소스 엔진입니다. 기본 루프는 다음과 같은 **Agentic Loop**로 동작합니다.

```text
┌────────────────────────────────────────────────────────────────┐
│                          [Agentic Loop]                        │
│                                                                │
│  ┌──────────────┐     ┌──────────────┐     ┌──────────────┐    │
│  │   1. Observe │ ──> │  2. Analyze  │ ──> │   3. Action  │ ─┐│
│  │ (HTML/스크린샷)│     │(LLM 다음행동)│     │(Playwright)  │  ││
│  └──────────────┘     └──────────────┘     └──────────────┘  ││
│          ▲                                                    ││
│          └────────────────────────────────────────────────────┘│
└────────────────────────────────────────────────────────────────┘
```

1. **관찰 (Observe)**: 현재 브라우저 페이지의 DOM 트리(HTML)를 간소화하고 스크린샷 이미지를 캡처하여 LLM의 입력값으로 전달합니다.
2. **분석 및 추론 (Analyze)**: LLM은 최종 목표에 도달하기 위해 현재 화면에서 취해야 할 다음 행동(예: '검색창을 찾아 Next.js를 타이핑해라', '검색 버튼을 클릭해라')을 판단합니다.
3. **행동 (Action)**: 판단된 결과를 JSON 형태로 파싱하여 Playwright와 같은 웹 브라우저 자동화 라이브러리를 통해 실제로 마우스를 클릭하고 키보드를 타이핑합니다.

## 3. 웹 에이전트 개발 시의 주요 도전 과제

* **토큰 효율화**: 전체 HTML 소스코드는 양이 너무 방대하므로, LLM에 던지기 전에 인터랙티브 요소(버튼, 링크, 입력 폼 등)만 추출하고 압축하는 DOM 트리 경량화가 필수적입니다.
* **상태 흐름 관리(State Preservation)**: 이전 행동에서 무엇을 했는지 기억하지 못하면 무한 루프에 빠지기 쉽습니다. AI의 대화 내역에 행동 이력(History)을 누적해 컨텍스트로 전달해야 합니다.
* **예외 복구 (Self-Correction)**: 지정한 요소를 브라우저가 찾지 못했을 때(Selector 에러 등), 에러 메시지를 LLM에 던져 다른 방식으로 시도하게 만드는 우회 복구 설계가 필요합니다.
