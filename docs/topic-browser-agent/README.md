# 📅 [1주차 Docs] `browser-use` 오픈소스 라이브러리 연동 및 구조 학습

이 문서에서는 깃허브에서 큰 인기를 끌고 있는 자율 웹 브라우징 에이전트 프레임워크인 **`browser-use`**의 공식 구조와 파이썬 환경에서의 활용 기술을 학습합니다.

* **연계 GitHub 오픈소스**: 🔗 [browser-use/browser-use GitHub Repository](https://github.com/browser-use/browser-use)

---

## 1. `browser-use` 란?

`browser-use`는 AI 모델이 마치 사람처럼 웹 브라우저를 조종할 수 있게 돕는 파이썬 전용 에이전트 프레임워크입니다. 밑바닥부터 Playwright를 조작하고 스크린샷 좌표를 분석하는 복잡한 과정들을 라이브러리 내부에서 처리해 주므로, 개발자는 **에이전트의 목표(Task)와 프롬프트 제어**에만 집중할 수 있습니다.

---

## 2. 핵심 클래스 아키텍처

`browser-use`는 크게 세 가지 핵심 구성 요소로 이루어져 있습니다.

```text
  ┌─────────────────────────────────────────────────────────┐
  │                        [ Agent ]                        │
  │  (최종 목표(Task) 정의 및 LLM 모델과의 추론 루프 실행)  │
  └──────────────┬───────────────────────────┬──────────────┘
                 │                           │
                 ▼                           ▼
        ┌─────────────────┐         ┌─────────────────┐
        │   [ Browser ]   │         │  [ Controller ] │
        │ (Playwright 기반 │         │ (커스텀 도구 정의│
        │ 브라우저 수명 제어)│         │  및 액션 제어)   │
        └─────────────────┘         └─────────────────┘
```

### 1) Agent
* 에이전트의 메인 엔진입니다.
* `task` (사용자의 자연어 지시 사항)와 `llm` (LangChain 기반의 LLM 모델 객체)을 인자로 받아, 내부적인 Agentic Loop를 실행합니다.

### 2) Browser & BrowserConfig
* Playwright 브라우저 인스턴스를 관리합니다.
* 헤드리스 모드(`headless=False`) 설정, 브라우저 창 크기, 특정 유저 데이터 디렉토리 로드 등을 구성할 수 있습니다.

### 3) Controller
* LLM이 기본 제공되는 동작(클릭, 입력, 페이지 이동 등) 외에 **커스텀한 도구(Python 함수)**를 실행할 수 있도록 연결해 주는 확장 모듈입니다.

---

## 3. 학습 및 검증 포인트

* **가상환경 격리**: 파이썬은 프로젝트별 패키지 충돌이 잦으므로, 반드시 가상환경(`.venv`)을 켜고 패키지를 설치하는 실무 습관을 들여야 합니다.
* **LangChain LLM 연동**: `browser-use`는 내부적으로 LangChain의 ChatModel 규격을 사용합니다. `langchain-openai`의 `ChatOpenAI` 객체를 생성하여 넘겨주는 연결 방식을 학습합니다.
* **토큰 소모량 체크**: 에이전트가 한 단계씩 이동할 때마다 현재 페이지의 HTML 스크린샷과 소스텍스트를 LLM에 전달하기 때문에, 과도한 시도 횟수(Limit)를 설정하면 토큰 비용이 급증할 수 있어 이를 제어하는 요령을 배웁니다.
